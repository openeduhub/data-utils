its_data.fetch.df_from_json_file(path: Path, columns: Iterable[str] | dict[str, str], prefix: str | None = '_source', key_separator: str = '.', filters: Collection[its_data.filters.Filter] = (), dropped_values: dict[str, Collection[its_data.data.Basic_Value_Not_None]] = {}, remapped_values: dict[str, dict[its_data.data.Basic_Value_Not_None, its_data.data.Basic_Value]] = {}, max_len: int | None = None) → DataFrame

Read the given line separated json file and turn it into a data frame.

Parameters:

  • path (Path) – The path to the json file to process.

  • columns (Iterable[str] | dict[str, str]) – The fields to keep from the json. If given as a dictionary, use the keys as the column names in the final data frame. Individual fields are split by the key separator for nested accesses.

  • prefix (str | None) – The fields to prefix any access with. Useful when all interesting data is contained within a nested object. Split by the key separator for nested accesses.

  • key_separator (str) –

    The sub string that denotes that we are accessing a nested json object.

    • Example: With “.”, “a.b” gets turned into a nested access of first field a and then field b.

  • filters (Collection[its_data.filters.Filter]) – Filter functions to be applied to individual entries of the json file before they are collected.

  • dropped_values (dict[str, Collection[its_data.data.Basic_Value_Not_None]]) – A mapping from field name to a collection of values that shall be dropped from entries with that field.

  • remapped_values (dict[str, dict[its_data.data.Basic_Value_Not_None, its_data.data.Basic_Value]]) – A mapping from field name to a dictionary that describes re-mappings of values within that field.

  • max_len (int | None) – The maximum number of entries to read from the json.

Returns:

The resulting data, as a pandas DataFrame.

Return type:

DataFrame

Download the latest data dump and save it to the given directory.

Parameters:

  • base_url (str) – The base URL from which to download the data.

  • target_file (str) –

    The file from the base URL to download.

    • Assumption: the file is located at base_url/target_file

  • output_dir (str | Path) – The directory in which to save the downloaded file.

  • output_file (str | Path | None) – The name of the file to download. By default, this is equal to target_file (without .gz, if applicable).

  • username (str | None) – The (Basic Auth) username to use for authorization.

  • password (str | None) – The (Basic Auth) password to use for authorization.

  • encoded_auth (str | None) – The already encoded username+password combination. If set, username and password are ignored.

  • skip_if_exists (bool) –

    • If set to True, skip download if the output file already exists.

    • If set to False, always override any already present files.

  • delete_compressed_archive (bool) – Whether to delete the compressed archive after it has been decompressed.

Returns:

The path to the (decompressed) downloaded file.

  • field_seq (Sequence[str]) – The sequence of keys to access in order to get to the field to modify.

  • to_drop (Collection[str | int | float]) –

    The values within the specified field to drop.

    • If the field contains a singular value to be dropped, it will be replaced by None.

    • If the field contains a list of values, all values to be dropped will be removed from the list.

    Note that dropping None is not supported, as this may result in unexpected behavior, since a list-value may not actually be represented by a leaf that contains a list of values. Because of this, None-values may also be introduced to lists of values.
